# python_job_spider
基于大数据的租房与招聘数据分析与可视化系统
项目简介
本项目是一款面向互联网行业毕业生及广大求职者的招聘与租房一体化数据分析与可视化系统，针对招聘与租房市场信息分散、数据质量良莠不齐、跨平台整合难的问题，融合网络爬虫、数据预处理、机器学习预测、多维度数据可视化技术，实现招聘 / 租房数据的统一采集、清洗、分析与可视化展示，并提供薪资智能预测功能，为求职者的就业选址、租房决策提供数据支撑。
系统整合拉勾网招聘数据、链家网租房数据，采用前后端分离架构开发，前端实现友好的可视化交互界面，后端完成数据处理与业务逻辑，结合 KNN、逻辑回归、朴素贝叶斯算法实现薪资等级预测，打破了传统招聘与租房系统分离的现状，实现信息的一体化整合与分析。

系统功能
本系统围绕招聘数据分析、租房数据分析、薪资智能预测、用户管理四大核心方向设计，功能模块清晰，交互友好，具体功能如下：
1. 用户信息管理模块
账号注册 / 密码加密登录，支持登录状态校验
个人信息编辑（邮箱、个人简介、居住地址、联系电话）
密码修改（三步式验证，保障账号安全）
2. 招聘数据可视化模块
数据概况：展示全量招聘信息，支持按学历要求、职位名称精准检索，分页展示详情
多维度可视化：从薪资、企业、福利、学历、融资、职位类型6 大维度，分全国 / 一线城市 / 新一线城市 / 二线城市展示数据
热门城市对比：对北京、上海、广州、深圳、杭州等 9 大热门城市，实现行政区、学历要求、公司规模、薪资待遇等维度的可视化对比
图表类型：柱状图、折线图、饼图、词云图、矩形树图、漏斗图、南丁格尔玫瑰图等
3. 租房数据可视化模块
数据概况：展示全量租房信息，支持按城市精准检索，分页展示房源详情（面积、朝向、楼层、价格等）
多维度可视化：从面积、楼层、朝向、价格4 大维度，分全国 / 一线城市 / 新一线城市 / 二线城市展示数据
城市房源详情：对热门城市实现行政区房源分布、面积分布、价格分布等精细化可视化
关联分析：实现房屋面积与价格的关联性可视化分析
4. 薪资智能预测模块
支持用户选择职位、公司规模、工作经验、学历、城市等特征
提供KNN、逻辑回归、朴素贝叶斯三种机器学习模型可选，预测对应岗位的最低薪资区间
实时返回预测结果，为求职者的薪资预期提供参考
5. 第三方网站入口
整合主流招聘网站（拉勾网等）、租房网站（链家网等）链接，支持一键跳转，实现深度信息检索。
系统架构
本系统采用分层架构设计，各层职责清晰，低耦合、高扩展性，整体架构分为五层：
前端 UI 层：基于 LayUI 实现，负责界面展示、用户交互、数据可视化渲染
应用层：基于 Flask 实现，负责接收前端请求、分发业务逻辑、返回处理结果
服务层：基于 Python 及数据分析 / 可视化库实现，负责数据处理、机器学习预测、可视化数据封装
数据层：包含网络爬取、数据清洗、数据分析、数据可视化、预测、用户管理六大模块，负责数据的全生命周期处理
数据库层：基于 MySQL 实现，负责用户信息、招聘数据、租房数据的结构化存储与管理
数据来源与处理
数据来源
招聘数据：2025 年 4 月拉勾网全国多职位招聘信息（含公司信息、职位要求、薪资、福利等）
租房数据：2025 年 5 月链家网全国多城市租房信息（含房源基本信息、面积、朝向、楼层、价格等）
数据处理流程
数据爬取：采用 Requests+PyQuery 编写爬虫，结合动态 User-Agent、Cookie 携带、随机请求延迟、Referer 伪造等反爬策略，实现数据稳定采集；
数据清洗：基于 Pandas 实现，包含缺失值处理、重复值去除、数据格式转换、分类特征数值化等；
数据预处理：针对薪资预测场景，将分类特征（公司规模、工作经验等）映射为数值特征，对薪资进行区间分桶，处理数据不平衡问题；
数据存储：爬取数据先保存为 CSV 文件，后批量写入 MySQL 数据库，实现结构化存储。
核心算法实现
本系统在薪资预测模块采用三种经典机器学习算法，实现薪资等级多分类预测，核心实现要点：
KNN 算法：选择 9 个最近邻居，采用距离加权投票（weights='distance'），暴力搜索计算距离；
逻辑回归算法：采用 One-vs-Rest（OVR）策略处理多分类，使用牛顿共轭梯度算法（solver='newton-cg'）优化；
朴素贝叶斯算法：基于贝叶斯定理与特征条件独立假设，实现轻量级高效分类。
模型评估
从准确率、精确率、召回率、F1 值四个维度对模型进行评估，模型整体性能满足基础应用需求，其中逻辑回归算法在多组测试中表现更优。

安装 MySQL 5.7+/8.0+，创建数据库lagou，并执行数据库表创建语句（见项目sql/目录）；
安装项目依赖：
bash
运行
pip install -r requirements.txt
配置修改
修改server.py中的 MySQL 数据库连接信息（用户名、密码、主机、端口）；
若需重新爬取数据，修改爬虫脚本（data.py/zufang_data.py）中的爬取配置（目标网站、城市、职位等）。

bash
